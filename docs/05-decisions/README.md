# Architecture Decision Records

Architecture Decision Records (ADR). Each significant architectural decision should be documented here using the ADR format. This provides a historical record of why decisions were made.

## Format

ADR files follow the naming convention:

```
ADR-XXXX-title.md
```

Where `XXXX` is a zero-padded sequential number (e.g., `ADR-0001-use-mongodb.md`).

## Template

Each ADR should contain the following sections:

- **Status** — Proposed, Accepted, Deprecated, or Superseded
- **Context** — The circumstances and forces that led to this decision
- **Decision** — The architectural decision that was made
- **Consequences** — The resulting impact, both positive and negative
